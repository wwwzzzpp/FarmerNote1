# 巡田记 v1 产品说明

## 目标

做一个自己随时打开、随时记一条巡田观察的手机 App。

核心体验只有一句话：

写一条记录，如果未来某个时间要处理，就顺手挂一个提醒，之后在时间线和待办里都能找到它。

## v1 范围

### 记录页

- 一个大文本输入框
- 一个“设置提醒”开关
- 打开后可选单次日期时间
- 一次保存，直接落本机

### 时间线页

- 按创建时间倒序展示所有记录
- 带提醒的记录显示状态：`待办 / 已逾期 / 已完成`
- 可以从时间线跳到对应待办

### 待办页

- 分成 `即将到来 / 已逾期 / 已完成`
- 每条待办支持 `完成 / 改时间 / 删除`
- 权限不足或同步失败时支持重试提醒/重试日历

## 明确不做

- 登录
- 多设备自动同步
- 图片上传
- 地块、天气、病虫害等结构化表单
- 重复提醒
- 真正闹钟式全屏唤醒
- 团队协作、统计报表、后台管理

## 关键体验规则

- 纯文字保存时，只创建记录
- 带未来时间保存时，同时创建一条单次待办
- 待办优先用 App 本地通知提醒
- 同时尝试把这条待办单向写入系统日历
- 如果用户选了过去时间，记录照样保存，但任务直接进逾期，不安排通知
- 提前完成未来任务时，取消本地通知，并删除未来的系统日历事件

## 数据策略

- 所有数据只保存在当前设备
- 不上传云端
- 不自动和另一台手机同步

## 后续扩展方向

- 图片记录
- 地块标签
- 农事分类
- 每天/每周重复巡田提醒
- 手动导出备份
